# Purpose

Define what value and capabilities the product must provide.

This artifact exists to translate the approved product representation into product requirements.

This artifact must not define:

- Product Structure
- UX Design
- Visual Design
- Technical Design
- MVP Definition
- Build Plan

These subjects belong to other artifacts.

# Acceptance Criteria

A valid Product Requirement must satisfy:

- Target users identified
- User problems identified
- User goals defined
- Product goals defined
- Required capabilities defined
- Success metrics defined
- Authority alignment defined
- Scope boundary defined
- No product structure defined
- No UX design defined
- No visual design defined
- No technical implementation defined
