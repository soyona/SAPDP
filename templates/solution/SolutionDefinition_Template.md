# Purpose

Define the solution approach for the validated problem.

This artifact exists to define:

- How the problem should be solved.
- Why the proposed approach is appropriate.
- What outcome the solution is expected to create.

This artifact must not define:

- Product Requirements
- Product Structure
- UX Design
- Visual Design
- Technical Design
- MVP Scope
- Build Plan

These subjects belong to later lifecycle stages.

A valid Solution Definition must satisfy:

- Solution clearly described
- Strategy defined

- No product design included

- No technical implementation included
- No MVP definition included
